☕ code ni compile chesinappudu yedaina error vaste irritate avtam kadha
☕ errors ni handle cheyyadaniki use ayye concept ye idhi
☕ Try Catch block Exceptions ni handle chestay. try block lo unna code without execute aythe ok, if error vaste exception ni throw chestundi catch block ah visiresina exception ni teeskoni meru mention chesina code ni execute chestundi
☕ oka example chooste meeke ardamaipotundi

JAVA


public class MyClass {


public static void main(String[] args) {


try {


int res = 1/0;


}


catch(Exception e){


System.out.println("okati takkuvaindi");


System.out.println(e);

}


}


}

output

okati takkuvaindi
java.lang.ArithmeticException: / by zero

☕ paina unna code lo error vaste particular message print ayyela manam cheyyochu. e anedi oka variable la which stores the type of error caused in try block
☕ anduke e ni print chesi nappudu manaki type of error em vacchindo print ayyindi, oka try block yenni catch blocks ayina undocchu
☕ we can mention type of error in catch block. yeppudaite ah mention chesina error vastundo appudu catch block lo unna code work avtundi. try and catch ki finally ane block kooda undi, errors tho relation lekunda ah finally block confirm ga execute avtundi

JAVA


public class MyClass {


public static void main(String[] args) {


try {


String name = null;


System.out.println(name.length());


}


catch(NullPointerException e){


System.out.println("error vacchindi nayana");


}


finally {


System.out.println("nekocchesindi");


}


}


}

output

error vacchindi nayana
nekocchesindi

☕ null ante null ye em ledu kali ani
