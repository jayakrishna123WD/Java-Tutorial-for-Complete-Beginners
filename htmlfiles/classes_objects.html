OOPS:

☕ Procedural programming lo functions data ni use cheskoni operations ni perform chestay, functions ki data store chese feature ledu.
☕ Object-Oriented Programming lo objects untay vatilo data and methods untay. functions ne methods antam oops terminology lo. Ikkada object ante yedo kaadu variable le kani difference yelantidi ante
☕ variable is like pen lo refill lantidi yenni sarlaina replace cheskovacchu, kani refill lo undedi only ink matrame inkem speciality undadu
☕ Object is like box la danilo pens, pencils, erasers, sharpener, scale ila chala untay (daniki oka structure untundi, data untundi, methods untay)

☕ mana body kooda oka class ye, nenu oka object ni body ane class ki meeru kooda oka object ye
☕ object is an instance of a class. ardam kaaleda detailed ga cheppukundam randi
☕ world lo yekkadaina deenini choodagane pendrive ane antaru because its structure defines that it was a pendrive alane class kooda anthe its like a template indulo attributes(variables) and methods(functions) untay.
☕ ee class ke manam object create chestam ante ippudu bank lo amount store cheyyalante meku oka account kavali ga ah account ye object ikkada bank ane class lo chaala methods untay like balance, withdraw, deposit, etc.. mana bank account use cheskoni ah methods ni access chestunnam kadha same concept ikkada kooda. so, pendrive class ki objects choddam
☕ ippudu nen 3 objects choopistunna vati names sony, sandisk and hp. each object ki seperate specifications untay like sony pendrive 16gb capacity, sandisk -- 64gb, hp -- 32gb. ee respective objects ni use chesi data ni store chestunnam class lo unna methods help tho like read method and write methods and folder attributes.
☕ inka clarity ga cheppalante class ki attributes, methods untay kani identity undadu same like mana pushpa raju ni police lu antaru kadha ala. identity lenodu class.
☕ if identity exists that should be called as object. like em leni kali pendrive class ki identity add cheste that will becomes sony, sandisk, hp pendrives. oka class ki yenni objects ayina create cheyyochu. there's no limit.
☕ final ga nen cheppedentante object is independent unit with unique identity, and it have characteristics like attributes, methods and identity
☕ class is like blue print, syllabus, description for an object. Object ante just oka variable lantidi anthe, methods and attributes undevi class lone. bank account use chesi bank services access chesi nattu, object use chesi class lo methods and attributes ni access cheskuntam anthe.
☕ Hope you understand the concept, ardam kakapote malli chadavandi okasari

☕ let's create object for pendrive class with attribute and print the size of pendrive.

JAVA


class Pendrive {


byte Size;


}


public class MyClass {


public static void main(String[] args) {


Pendrive sony = new Pendrive();


Pendrive sandisk = new Pendrive();


Pendrive hp = new Pendrive();


sony.Size = 16;


sandisk.Size = 64;


hp.Size = 32;


System.out.println(sony.Size);


System.out.println(sandisk.Size);


System.out.println(hp.Size);


}


}

64
32
